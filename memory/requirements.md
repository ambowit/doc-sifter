# 尽职调查文档整理系统（DD Organizer）

## 产品语言

中文

## 要构建什么

一个专业级的法律尽职调查文档整理系统，为律师事务所、投行和企业法务提供自动化的尽调资料整理能力。系统允许用户上传尽调报告样本结构和尽调资料 Zip 包，通过 AI 自动解析、映射和整理，最终输出专业、可交付、可追溯的尽职调查报告。核心价值在于「证据驱动」和「可追溯性」，系统不允许 AI 编造任何事实，所有内容必须基于已上传的文件。

## 目标受众

综合性专业团队，涵盖律师事务所尽调团队（处理并购、投融资项目）、投行分析师（快速整理尽调资料）、企业法务部门（内部合规与风控）。这些用户的共同特点是需要处理大量结构化文档，对数据准确性和可追溯性要求极高，追求专业、严谨、高效的工作流程。

## 设备支持

优先桌面端（1440px），后续扩展移动端支持。专业法律系统的主要使用场景在桌面端。

## MVP 范围

第一版实现核心流程4个页面：项目列表 + 文件上传 + 章节映射 + 报告预览。后续迭代扩展至完整7页。

## 功能与页面

### 核心页面结构（9个页面）

1. **尽调项目列表页（Dashboard）**：展示所有尽调项目的概览，包括项目名称、客户/标的公司、当前阶段（未上传/解析中/可生成/已生成）、最近更新时间，支持新建项目和进入项目详情。
2. **新建尽调项目页**：表单页面，收集项目基本信息（项目名称、项目类型、尽调报告语言、是否启用严格证据模式等）。
3. **文件上传页（双输入核心页）**：分为两个区块 - 区块A上传尽调报告样本结构（PDF/Word），区块B上传尽调资料Zip。上传后展示文件解析状态、目录结构树和文件统计信息。
4. **解析与映射状态页**：实时展示解析进度的可视化流程条和日志输出，包括样本结构解析、尽调资料解析、章节映射、风险与缺失检测等步骤。
5. **章节映射审阅页（核心价值页）**：左栏显示尽调报告树状结构和每个章节的匹配状态，右栏显示当前章节关联的证据文件列表和关键摘要，支持拖拽文件调整映射关系。
6. **AI 总结预览页**：展示每个章节的已审阅文件清单和基于证据的事实性总结，明确标注风险/异常和资料缺失部分。
7. **报告生成与导出页**：提供输出格式选择（Word/PDF）、引用附录开关、在线预览（可点击查看来源文件）和下载功能。增加证据引用侧栏，展示每章节的证据文件、摘要和置信度，支持一键定位原文。
8. **模板指纹页（Template Fingerprint）**：从样本 DD 报告中提取并固化模板结构和样式。包含三个子模块：目录结构预览（显示章节层级、固定内容标记）、样式规范预览（字体、段落、列表、表格、图表样式）、引言编辑（可编辑变量如标的公司名、委托方、截止日等）。
9. **定义与简称管理页（Definitions）**：管理报告中使用的主体简称列表。支持增删改查、类型筛选（公司/自然人/机构/交易/其他）、简称冲突检测和来源文件引用。每个定义项包含简称、全称、类型、来源文件和备注。

### 核心功能特性

- **证据驱动原则**：所有 AI 生成的内容必须基于已上传文件，未发现资料的部分明确标注，不允许 AI 补充事实。
- **双输入机制**：同时上传样本报告结构和尽调资料，通过智能映射建立对应关系。
- **可追溯性**：每段内容都可以追溯到源文件，支持点击查看来源。
- **多角色测试账号**：登录页提供 4 种角色的真实测试账号（管理员、高级律师、初级律师、助理），点击即可自动初始化并登录，数据保存在数据库中。
- **压缩包文件支持**：文件上传支持 PDF、Word、Excel、ZIP/RAR/7Z 压缩包格式，单文件最大 500MB。通过 SuperunStorage 实现真实的文件上传与存储，支持实时进度显示。
- **ZIP 压缩包自动解压**：上传 ZIP 文件时自动在前端解压，提取内部文件并逐个上传。显示解压进度和当前文件名，自动过滤隐藏文件和系统文件。RAR/7Z 格式需要用户本地解压后上传。
- **AI 智能生成报告（全自动）**：用户上传数据室文件后，无需手动映射文件到章节。AI 自动阅读所有文件的 OCR 内容，根据每个章节标题智能匹配相关证据，自动生成专业的尽调报告内容。简化工作流程为：模板指纹 → 数据室文件 → 一键生成报告。AI 会智能分类文件（公司基本信息、股权结构、知识产权、劳动人事等），并为每个章节生成内容、发现的问题、风险评估和处理建议。
- **AI 智能生成报告结构**：上传模板文件时，AI 自动分析并生成专业的尽调报告章节结构。即使无法提取文件内容（如 PDF/Word），AI 也会根据文件名和行业最佳实践生成 8-10 个专业章节，每个章节包含 2-4 个子章节，并提供详细的核查要点说明。支持"AI 生成结构"一键生成。
- **OCR 文字提取**：文件上传后自动使用 AI 视觉能力（Gemini）提取图片和 PDF 中的文字内容，存储在数据库中。支持的格式包括 JPEG/PNG/GIF/WebP 图片和 PDF 文档。提取的内容包括完整文字、内容摘要和关键实体（公司名称、人名、日期、金额、百分比等）。文件列表显示 OCR 状态指示器（未提取/提取中/已提取），支持批量手动提取。报告生成时优先使用 OCR 提取的内容，提高股权结构和报告内容的准确性。
- **品牌视觉**：采用企业 Logo 的深海军蓝 + 青柠绿配色，侧边栏支持伸缩切换。
- **股权结构图**：可视化展示股东、目标公司和子公司的多层级股权关系。支持缩放查看、节点编辑（添加/修改/删除股东）、持股比例警告和导出功能。根据股东类型（自然人/法人/持股平台）显示不同样式。
- **设置页面**：支持个人信息编辑、外观主题切换（浅色/深色/跟随系统）、通知设置、报告偏好、管理员专属设置。
- **帮助中心**：包含FAQ常见问题解答、快速入门指南、核心特性说明、技术支持联系方式。
- **数据持久化**：使用 SuperunCloud 数据库存储项目、文件、章节和映射数据。所有操作前验证项目存在性，防止数据丢失和孤儿文件。
- **测试账号初始化**：Edge Function 自动初始化测试账号，首次点击时创建，后续直接登录。

### 设计原则

- **UI 风格**：专业、克制、法律行业风格，采用深海军蓝(#1a365d)+青柠绿(#c4d82e)品牌色，类似投行和律所系统。
- **交互原则**：AI 不主动补充事实，所有内容可追溯，缺失即提示。
- **技术约束**：前端展示为主，使用 Mock API，不实现真实后端逻辑。

## 技术特性

- 文档解析与映射可视化
- 树状结构交互（章节与文件关联）
- 实时解析状态展示
- 拖拽式映射调整
- 证据追溯系统
- 模板样式学习与复刻
- 定义与简称自动生成
- 股权结构图可视化编辑
- **三层JSON数据模型**：Template Fingerprint（样本指纹）+ Knowledge Graph（知识图谱）+ Report Plan（报告计划）
- **DOCX生成引擎**：基于三层数据模型的专业Word文档生成，支持完整样式控制，已集成到报告预览页面的"导出报告"功能
- **报告生成引擎（投资尽调专业级）**：Edge Function (generate-report) 采用投资方视角的专业模板，重点揭示投资风险和交易文件设计建议。风险覆盖范围包括：注册资本实缴、关键人员合规（兼职审批、禁止兼任）、体外持股/任职、知识产权权属、重大合同履约、内部治理制度、ESOP等。建议格式关联交易文件设计（交割条件、业务里程碑、交割后义务、投资方特殊权利、退出保障）。内容要求800-1500字，包含具体数据和表格
- **定义管理功能（已修复）**：定义与简称页面现在使用真实数据库存储。新建 definitions 表存储简称、全称、类型、来源文件等信息，支持 AI 自动提取功能从数据室文件中识别主体名称
- **项目数据持久化优化**：修复了多个数据持久化问题：1) 退出登录功能使用 async/await 确保完整执行；2) useCurrentProject 只在明确的"未找到"错误(PGRST116)时才清除 localStorage，避免因临时网络问题导致项目ID被误删；3) 登出时保留报告数据（report_前缀），因为已有 RLS 保护；4) 登入时自动刷新项目查询，登出时清除 React Query 缓存

## 数据模型架构

### Layer 1: Template Fingerprint（样本指纹）

从样本DD报告中提取的完整模板描述，包括：

- 编号配置（NumberingConfig）
- 页面设置（PageConfig）
- 样式集（h1-h4、body、quote、caption、footnote）
- 列表样式、表格样式、图表样式
- 目录结构（TOCItem）
- 章节蓝图（SectionBlueprint）
- 引言变量和内容模板

### Layer 2: Knowledge Graph（数据室知识图谱）

从尽调资料中提取的结构化知识：

- 实体（Entity）：公司、股东、个人、机构、交易、合同、资产、知识产权
- 事实（Fact）：股权持有、公司设立、治理、合同、诉讼、财务、税务等
- 文档（Document）：带解析状态的文件元数据
- 章节覆盖率分析和冲突摘要

### Layer 3: Report Plan（报告编排计划）

可渲染的最终报告结构：

- 章节计划（SectionPlan）：包含内容块数组
- 内容块类型：段落、表格、定义表、图表、列表、问题表、缺失提示等
- 报告元数据和统计信息
- 附录配置

10. **2025-02-13 修复 - 切换账号死循环**:
  - **问题**：点击律师测试账号登录后进入死循环
  - **根本原因**：useAuth 是普通 hook，每个组件调用都创建独立状态和监听器（Login、ProtectedRoute、AppLayout 各一份）。多个实例之间认证状态不同步，导致 Login 认为已登录跳转到 /，但 ProtectedRoute 认为未登录又跳回 /login，循环往复
  - **修复**：
  1. 将 useAuth 重构为 AuthContext + AuthProvider 模式，全应用共享同一份认证状态
  2. AuthProvider 放在 RootLayout 内，作为所有路由的父组件
  3. 移除 useAuth 内部的 useNavigate，导航逻辑交给各组件自行处理
  4. Login 页面统一用 useEffect 监听 isAuthenticated 来跳转，避免多重导航冲突

  - **实现**：1) 新增 `ocrFailedIds` 状态跟踪提取失败的文件；2) 失败文件显示红色“提取失败·重试”徽章（可点击单独重试）；3) 未提取文件显示可点击的“未提取”徽章（点击触发提取）；4) 文件列表上方显示失败横幅（红色）和未提取横幅（琥珀色），带“全部重试”或“全部提取”按钮；5) “全部提取”按钮点击后立即显示加载状态和 toast 通知，避免无反馈问题
13. **2026-02-18 修复 - 报告生成超时重试**:
  - **问题**：生成报告时第4批AI调用返回504 Gateway Timeout，导致整个报告生成失败，已成功的前3批内容也丢失
  - **修复**：为每个批次添加最多2次自动重试机制，仅对超时类错误（504/timeout/Gateway）自动重试，每次重试前等待3秒，进度条显示重试状态
14. **2026-02-18 增强 - 报告生成断点续传**:
  - **需求**：已成功的批次保留，失败的可重试，直到全部成功
  - **实现**：1) 新增 `batchResults` Record + `failedBatchIndices` Set 状态跟踪每批结果；2) 批次失败后标记为 error 但继续处理后续批次（不中断）；3) 按钮变为“重试失败批次”，点击仅重试失败的批次；4) 部分失败时保存已完成内容为临时报告，提供“查看已生成内容”入口；5) 提供“从头开始”选项清除缓存重新生成；6) 全部成功后自动清理缓存并跳转预览
